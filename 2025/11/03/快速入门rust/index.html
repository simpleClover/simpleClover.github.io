2. 试试Fibonacci

如果是Oier，估计对这个东西已经熟悉得不能再熟了吧：
Fib(n)是这样的一个数列，在n=0或n=1时Fib(n)=n，在n大于1时，Fib(n) = Fib(n-1) + Fib(n-2)。

我们定义了一个新函数fib，函数需要一个i32类型的参数n，返回类型也为i32，可以注意到即使fib定义在main后面也可被main调用，因为rust并不在乎函数在哪里被定义。

for i in 1..=5是一个循环，1..=5作为一个可迭代对象包括了1,2,3,4,5，若改为1..5则只包括1,2,3,4。

读者可能好奇fib函数中最后一行的表达式fib(n - 1) + fib(n - 2)是什么，其实在rust中函数最后一个表达式的结果就是函数返回值，注意这里说的是表达式，也就是说如果不小心写成fib(n - 1) + fib(n - 2);（多了个分号导致表达式变成语句）就要报错了，当然使用return提前返回也是没问题的。

在rust中定义变量使用let关键字，如 let x2 = 20i32，rust很聪明以至于在大部分时间它可以自动推断出变量的类型是什么，不过我们也可以手动定义变量类型，如let x1: i32 = 10;，在rust中整数的默认类型是i32，所以10、10_i32、10i32其实是一样的，浮点数的默认类型为f64。

对于使用 let定义的变量，其默认是不能变的变量（很奇怪对吧，但它就是这样的），如果想要让变量在后续可以改变，那么需要加上 mut关键字，如 let mut div，rust支持声明重名变量，同时后声明的变量将覆盖先声明的变量，上面代码中的俩个x1实际上是相同名字的不同变量。

4. 所有权是什么

想必来学rust的朋友在正式接触rust前或多或少听说过rust的所有权和内存安全。所有权是rust用于管理内存的一套规则，我们知道程序在运行的时候都要使用内存，而rust就是通过所有权系统保证了程序运行的内存安全。

所有权有三条规定：

  1. 每个值在rust中都有它的所有者
  2. 同一时间一个值只有一个所有者
  3. 当所有者离开作用域时值就会被丢弃

在给出所有权的例子前，我们先把rust中的类型分为两类，一类实现了Trait Copy，一类没有实现Trait Copy（一个Trait在rust中是一组特性，实现某个Trait即这个类型实现了这个Trait所需的特性）

对于实现了Trait Copy的类型，把值赋值给另外一个变量的行为等价于先克隆自己，然后将克隆得到的值的所有权转移给另外一个变量，此时两个变量分别掌握内存中的两份值，每个值都有自己的所有者

对于没有实现Trait Copy的类型，把值赋值给另外一个变量的行为会导致所有权从自身移动到另外一个变量，此时原变量失去所有权，无法通过原变量访问内存中的值

5. 我们需要更多类型！

我们知道（前面讲过吗，没有吧！）rust有一些基本类型
整数型：i8，u8，i16，u16，i32，u32，i64，u64，i128，u128，isize，usize
浮点型：f32，f64
布尔型：bool
字符型：char

其中特殊的有几个：
isize和usize的大小与架构有关，可能为32位也可能为64位
f32和f64遵守IEEE754，因此除了一般数字外还有其对应的NaN和INFINITY
char类型在rust中拥有四字节大小
